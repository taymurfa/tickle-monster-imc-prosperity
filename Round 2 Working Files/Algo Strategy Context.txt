3. PRODUCT STRATEGIES

═══ OSMIUM — Mean-Reverting Market Maker ════════════════════════════════════

Structure: Classic inventory-aware market making with EMA fair value.

Key mechanics:
  1. MICROPRICE EMA: Fair value = EMA of microprice (volume-weighted mid).
     Microprice = (best_bid * ask_vol + best_ask * bid_vol) / (bid_vol + ask_vol)
     This is more accurate than simple mid since it reflects order imbalance.

  2. QUADRATIC SKEW RESERVATION PRICE:
     reservation = ema - linear_adj - quad_adj
     linear_adj = position * OSMIUM_SKEW_COEFF (0.01) — small lean
     quad_adj = sign(pos) * (pos/limit)^2 * OSMIUM_SKEW_QUAD_MUL (10.0)
     — At position=80 (limit): quad_adj = 10, strongly pushes quotes away from
       extending inventory further.
     — At position=40 (50% limit): quad_adj = 2.5, mild lean.
     — Quadratic gives soft constraint that smoothly increases.

  3. INVENTORY THROTTLE:
     Beyond THROTTLE_THR (60), taker edge increases to avoid extending inventory.
     excess = max(0, |pos| - 60)
     buy_edge  = TAKER_EDGE + excess * 0.5  (more expensive to go longer)
     sell_edge = max(0.1, TAKER_EDGE - excess * 0.3)  (cheaper to reduce)

  4. TAKER FILLS: Take immediately if price crosses reservation ± taker_edge.
     Maximum 10 units per tick.

  5. PASSIVE QUOTES: Post bid at reservation - spread_offset, ask at + spread_offset.
     Adaptive: if spread > 4 ticks, tighten offset to 1.5 (more aggressive).
     Maximum 12 units per passive quote.

Final OSMIUM parameters (confirmed optimal via sweep):

  OSMIUM_TAKER_EDGE             = 1.0
  OSMIUM_SKEW_QUAD_MUL          = 10.0
  OSMIUM_QUOTE_SIZE             = 12
  OSMIUM_TAKE_SIZE              = 10
  OSMIUM_SPREAD_OFFSET          = 2.5
  OSMIUM_SKEW_COEFF             = 0.01
  OSMIUM_INVENTORY_THROTTLE_THR = 60


═══ PEPPER — Directional Trend-Following Accumulator ════════════════════════

Structure: Linear trend model + tiered position-building accumulator.

The core insight: PEPPER price rises at a CONSTANT SLOPE of 0.001 per timestamp.
This was confirmed by linear regression on all 3 days of R2 data. If you hold
a long position, you capture that rising price passively.

Price model:
  fair(t) = base_ema + TREND_SLOPE * t
  base_ema ≈ implied intercept for each day (updated via slow EMA)
  implied_base = current_mid - TREND_SLOPE * current_timestamp

Goal: Build a long position of ~79 (just below the 80 hard limit) ASAP,
then hold and capture the trend. Each additional unit held gains ~1 XIREC
per 1,000 timestamp units. Over 30,000 timestamps (3 days), that's 30
XIRECs/unit * 79 units = 2,370 XIRECs from trend alone (rough estimate).

Tiered accumulation:
  Phase 1 (pos < 60):   Cross the spread aggressively (taker) — build fast
  Phase 2 (60 ≤ pos < 79): Post passive bids at best_bid — earn spread too
  Phase 3 (pos ≥ 79):   Hold; only react to opportunistic signals

Opportunistic signals:
  BAD ASK: Someone asks at ≤ fair - 5.0 → buy all available immediately
  BAD BID: Someone bids at ≥ fair + 8.0 → sell all available immediately
  Emergency: If at hard limit (80), immediately sell 1 unit at best_bid

Positions carry over day-to-day in backtest. So position built on day -1
persists into day 0, etc. This is critical — PEPPER accumulation on day -1
compounds through all 3 days.

Final PEPPER parameters (all confirmed optimal at R1 values):

  PEPPER_BASE_EMA     = 0.002
  PEPPER_TARGET_POS   = 79

4. MARKET ACCESS FEE (MAF) — bid() METHOD
────────────────────────────────────────────────────────────────────────────────

Mechanism: Each trader submits a bid (integer XIRECs) once per round.
Highest bidder(s) unlock +25% extra order book volume. The extra quotes
appear at the SAME prices and follow the SAME distribution as existing quotes
— just 25% more volume.

Implementation in Trader class:
  def bid(self) -> int:
      return 1350

Reasoning for 1350:
  - MAF EV calibrated using R1 algo leaderboard data scaled to R2:
      * Our R2 algo scores ~24k on website scale (3 days)
      * Extra 25% volume adds ~1,800 XIRECs (OSMIUM only; PEPPER already maxed)
      * Break-even bid: 1,800. Any bid below is positive EV if accepted.
  - Top 50% of ~4,000 expected bids accepted. Median bidder (rank 2,000) has
    EV ~1,651 and likely bids 60-70% of EV = ~991-1,156.
  - Naive cluster at exactly 1,000 and 1,200 is large (round-number bias).
  - 1,350 beats that cluster and puts us roughly in top 200-300 of 4,000 bids.
  - Net EV if accepted: 1,800 - 1,350 = +450 XIRECs.
  - Non-round number to beat anyone bidding exactly 1,300.

Note: The backtest does NOT simulate extra MAF volume, so the 296,134 PnL
figure does NOT include any MAF benefit — it's pure base strategy. The MAF
upside is additional.

11. LESSONS LEARNED / GOTCHAS
────────────────────────────────────────────────────────────────────────────────

1. EMA ALPHA SCALING: When tick density changes, EMA alpha must scale
   proportionally. alpha_new = alpha_old * (ticks_old / ticks_new).
   R1→R2: 0.05 * (500/10000) = 0.0025 → we used 0.001 (even slower, confirmed better).

2. SWEEP SCRIPT CORRUPTION: Early sweep scripts used in-place file modification
   without proper try/finally restore. This caused the strategy to be left in
   a mutated state, making subsequent sweeps show incorrect baselines. Always
   use restore() at both end AND in a try/finally block.

3. UNICODE IN PRINT: Using Unicode characters (like box-drawing ─) in print()
   causes UnicodeEncodeError on Windows with cp1252 encoding. Use ASCII only
   or set PYTHONIOENCODING=utf-8.

4. BACKTEST NOISE FLOOR: On R2 data (3 days * 10k ticks), the PnL noise floor
   is approximately ±35-200 XIRECs. Do not adopt changes smaller than ~300.

5. PEPPER TARGET_POS IS CRITICAL: Reducing from 79 to 75 costs ~5,758 PnL.
   Reducing to 60 costs ~33,010. The trend slope compounds — every unit of
   max position * every timestamp * 0.001 = real money. Never reduce this.

6. POSITIONS CARRY OVER IN BACKTEST: Days -1 → 0 → 1 are continuous.
   Position at end of day -1 becomes starting position of day 0. This is
   why PEPPER works well — the long position built on day -1 earns all of
   days 0 and 1 as well.

7. MAF BID DOESN'T AFFECT BACKTEST: The backtest does not simulate the extra
   25% volume from winning the MAF bid. So all PnL figures are conservative
   lower bounds — actual live PnL should be higher if MAF is won.

8. GIT .PYC FILES: dashboard/__pycache__/backtest_engine.cpython-314.pyc gets
   staged automatically with `git add dashboard/`. Always `git restore --staged`
   it before committing.
